# 需求文档

## 功能需求

### 1. 用户账户与个人信息管理

- **FR1 用户注册与登录**：系统应允许用户注册账户、登录账户和退出登录。
- **FR2 个人资料管理**：系统应允许用户填写和修改个人基本信息，如昵称、性别、年龄、身高、体重等。
- **FR3 健康目标设置**：系统应允许用户设置个人健康目标，例如每日步数目标、饮水目标、睡眠目标、体重管理目标等。

### 2. 健康数据记录功能

- **FR4 睡眠记录**：系统应允许用户记录睡眠相关信息，包括入睡时间、起床时间、睡眠时长、睡眠质量等。
- **FR5 运动记录**：系统应允许用户记录运动类型、运动时长、运动日期以及运动消耗的能量。
- **FR6 饮食记录**：系统应允许用户记录每日饮食信息，包括食物名称、摄入时间、摄入热量等。
- **FR7 压力记录**：系统应允许用户记录压力水平，例如低、中、高，或通过评分方式记录压力状态。
- **FR8 步数记录**：系统应允许用户录入或同步每日步数信息。
- **FR9 饮水记录**：系统应允许用户记录每日饮水次数和饮水总量。
- **FR10 生理周期记录**：系统应允许用户记录生理周期开始时间、结束时间、周期长度及相关备注信息。

### 3. 健康数据展示与分析功能

- **FR11 数据查询**：系统应允许用户按日期查看历史健康记录。
- **FR12 数据统计**：系统应对用户的健康数据进行统计，如每日、每周、每月的睡眠时长、饮水量、步数、热量摄入、运动时长等。
- **FR13 可视化展示**：系统应以图表或列表形式展示健康数据变化趋势，帮助用户直观了解自身健康状况。
- **FR14 健康状态总结**：系统应根据用户记录的数据生成阶段性健康总结，例如"本周平均睡眠时长""本月饮水达标天数"等。

### 4. 提醒与反馈功能

- **FR15 饮水提醒**：系统应支持按照设定时间提醒用户饮水。
- **FR16 睡眠/运动/记录提醒**：系统应支持提醒用户按时睡觉、运动或填写当天健康信息。
- **FR17 目标完成反馈**：当用户达到每日或阶段性健康目标时，系统应给予提示或反馈。

### 5. 智能问答功能

- **FR18 健康问题问答**：系统应允许用户以自然语言输入问题，并返回相应回答。
- **FR19 数据相关问答**：系统应支持用户询问与自己记录数据相关的问题，例如"我这周平均睡眠多久""最近饮水是否达标"。
- **FR20 基础健康建议**：系统应根据用户输入的问题和记录情况，提供一般性的健康管理建议。
- **FR21 风险提示与边界说明**：对于医疗诊断类、高风险类问题，系统应提示"本系统仅提供一般健康管理建议，不能替代专业医生诊断"。

### 6. 数据管理功能

- **FR22 数据修改与删除**：系统应允许用户对已记录的健康数据进行修改和删除。
- **FR23 数据备份或导出**：系统应支持用户导出自己的健康数据，便于保存和查看。

## 非功能需求

### 1. 易用性需求

- **NFR1 界面简单直观**：系统界面应简洁清晰，主要功能入口明显，便于普通用户快速上手。
- **NFR2 操作便捷**：用户完成一次健康数据录入的操作步骤不宜过多，常用记录功能应尽量在较少步骤内完成。
- **NFR3 易学习**：新用户在短时间内应能够理解系统基本功能并独立完成记录操作。

### 2. 性能需求

- **NFR4 响应时间合理**：在正常网络条件下，页面切换、数据查询和普通问答响应应在可接受时间内完成。
- **NFR5 支持日常并发访问**：系统应能够满足课程项目范围内多用户正常使用，不因少量并发而频繁卡顿或崩溃。

### 3. 可靠性需求

- **NFR6 数据保存可靠**：用户提交的健康记录应被正确保存，避免无故丢失。
- **NFR7 异常处理**：当网络异常、输入错误或系统故障时，系统应给出提示信息，而不是直接报错或退出。

### 4. 安全与隐私需求

- **NFR8 用户隐私保护**：系统应保护用户的个人信息和健康数据，避免未经授权的访问。
- **NFR9 账户安全**：系统应要求用户登录后才能查看和修改个人健康数据。
- **NFR10 敏感信息谨慎展示**：涉及生理周期、压力等较敏感数据时，系统应仅对本人可见。

### 5. 可维护性需求

- **NFR11 模块化设计**：系统应采用较清晰的模块划分，如用户模块、记录模块、统计模块、问答模块，便于后续维护和扩展。
- **NFR12 易于扩展**：后续若要增加新健康指标或新的智能分析功能，系统应便于修改和扩展。

### 6. 兼容性需求

- **NFR13 多终端适配**：系统应至少支持常见浏览器或常见移动端界面访问。
- **NFR14 基本兼容性**：系统在主流设备和分辨率下应保持正常显示与基本功能可用。

### 7. 合规与伦理需求

- **NFR15 非医疗诊断声明**：系统提供的信息应定位为健康管理参考，不应冒充医学诊断结论。
- **NFR16 建议内容适度**：系统不应输出明显危险、夸大或不负责任的健康建议。
